# SJ Wiki Dashboard

Dynamic web dashboard for the Heilind / ICC knowledge base wiki. Reads your Obsidian vault's frontmatter and renders KPI cards, org charts, source timelines, and people data as a hosted web app.

## Stack

- **Next.js 14** (static export → GitHub Pages)
- **Recharts** — bar charts for source timeline and people by BU
- **react-organizational-chart** — interactive org tree
- **Tailwind CSS** — dark theme
- **Python 3.9+** — vault parser (`parse_vault.py`)

---

## Setup on Mac Mini

### 1. Prerequisites

```bash
# Node 20+
node --version

# Python 3.9+ with pyyaml
pip3 install pyyaml

# npm dependencies
cd sj-wiki-dashboard
npm install
```

### 2. Configure vault path

By default `parse_vault.py` looks for the vault at the hardcoded path in the script. Override with an env var:

```bash
export VAULT_PATH="/path/to/your/SJ_AI_Brain"
```

Or edit `parse_vault.py` line near the top:

```python
VAULT_PATH = os.environ.get("VAULT_PATH", "/your/actual/vault/path")
```

### 3. Parse the vault

```bash
python3 parse_vault.py
# Writes: app/data/wiki_data.json
```

### 4. Run locally

```bash
npm run dev
# Open: http://localhost:3000
```

### 5. Build for static deployment

```bash
npm run build
# Output: ./out/  (static HTML/CSS/JS — no server required)
```

## Deployment: GitHub Pages

### One-time setup

1. Push this repo to GitHub
2. Go to **Settings → Pages → Source → GitHub Actions**
3. That's it — the workflow in `.github/workflows/deploy.yml` auto-deploys on every push to `main`

### If your repo lives at a subpath (e.g. `username.github.io/repo-name`)

Edit `.github/workflows/deploy.yml`:

```yaml
env:
  NEXT_PUBLIC_BASE_PATH: "/your-repo-name"
```

And update `next.config.ts`:

```ts
basePath: process.env.NEXT_PUBLIC_BASE_PATH || "",
```

(Already wired — just set the env var.)

## Update workflow (weekly or whenever you ingest new sources)

```bash
# 1. Re-parse the vault
python3 parse_vault.py

# 2. Verify the output looks right
cat app/data/wiki_data.json | python3 -m json.tool | head -50

# 3. Commit and push — GitHub Actions auto-deploys
git add app/data/wiki_data.json
git commit -m "chore: refresh wiki data $(date +%Y-%m-%d)"
git push
```

Or set up a cron job on the Mac Mini to run steps 1–3 automatically each night.

## Troubleshooting

| Problem | Fix |
|---|---|
| `ModuleNotFoundError: yaml` | `pip3 install pyyaml` |
| Org chart shows no tree | All people missing `reports_to` in frontmatter — check parser output |
| GitHub Pages 404 | Confirm `trailingSlash: true` in `next.config.ts` |
| `NEXT_PUBLIC_BASE_PATH` not taking effect | Rebuild after setting env var; env vars bake in at build time |
| Blank JSON output | Verify VAULT_PATH is correct and `wiki/` folder exists at that path |
